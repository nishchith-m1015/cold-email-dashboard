# Root-Level Files Analysis

## Overview
This document categorizes all root-level files in the project, explaining which files **MUST** stay in root (due to tooling requirements) vs which **CAN** be moved to subdirectories.

---

## üî¥ MUST Stay in Root (Tooling Requirements)

These files are **hardcoded** by their respective tools to be in the project root. Moving them will break functionality.

### 1. **package.json** & **package-lock.json**
- **Why**: npm/Node.js requires these at root
- **Tool**: npm, yarn, pnpm
- **Can Move?**: ‚ùå NO - npm looks for `package.json` in current directory
- **Location**: Root only

### 2. **next.config.js**
- **Why**: Next.js automatically loads `next.config.js` from root
- **Tool**: Next.js framework
- **Can Move?**: ‚ùå NO - Next.js only checks root directory
- **Location**: Root only
- **Note**: Can be `next.config.ts` or `next.config.mjs` but still must be in root

### 3. **tsconfig.json**
- **Why**: TypeScript compiler looks for `tsconfig.json` in root by default
- **Tool**: TypeScript compiler
- **Can Move?**: ‚ö†Ô∏è PARTIAL - Can use `extends` to reference a config in a subfolder, but the root `tsconfig.json` is still needed
- **Location**: Root (can extend from subfolder)

### 4. **middleware.ts**
- **Why**: Next.js middleware **must** be at root level
- **Tool**: Next.js App Router
- **Can Move?**: ‚ùå NO - Next.js convention: `middleware.ts` at root intercepts all routes
- **Location**: Root only
- **Purpose**: Route protection, authentication, redirects

### 5. **vercel.json**
- **Why**: Vercel deployment platform looks for this at root
- **Tool**: Vercel CLI/platform
- **Can Move?**: ‚ùå NO - Vercel only checks root
- **Location**: Root only
- **Purpose**: Deployment configuration, cron jobs, redirects

### 6. **.gitignore** & **.gitattributes**
- **Why**: Git looks for these dotfiles at repository root
- **Tool**: Git version control
- **Can Move?**: ‚ùå NO - Git only reads root-level dotfiles
- **Location**: Root only

### 7. **.npmrc**
- **Why**: npm reads `.npmrc` from root (or user home)
- **Tool**: npm package manager
- **Can Move?**: ‚ùå NO - npm only checks root or `~/.npmrc`
- **Location**: Root only
- **Current Content**: `legacy-peer-deps=true`

### 8. **next-env.d.ts**
- **Why**: Auto-generated by Next.js TypeScript integration
- **Tool**: Next.js TypeScript plugin
- **Can Move?**: ‚ùå NO - Auto-generated, Next.js controls location
- **Location**: Root only (auto-generated)

## üü° CAN Be Moved (With Configuration Changes)

These files can be moved, but require updating tool configurations.

### 1. **jest.config.ts**
- **Why**: Jest can be configured to look elsewhere
- **Tool**: Jest test runner
- **Can Move?**: ‚úÖ YES - Can move to `config/jest.config.ts` and update `package.json`:
  ```json
  "scripts": {
    "test": "jest --config config/jest.config.ts"
  }
  ```
- **Recommendation**: Keep in root (standard convention)

### 2. **playwright.config.ts** & **playwright.config.setup.ts**
- **Why**: Playwright can accept custom config path
- **Tool**: Playwright E2E testing
- **Can Move?**: ‚úÖ YES - Can move to `e2e/playwright.config.ts` and update command:
  ```bash
  npx playwright test --config e2e/playwright.config.ts
  ```
- **Recommendation**: Move to `e2e/` folder (makes sense organizationally)

### 3. **postcss.config.js**
- **Why**: PostCSS can be configured via `package.json` or custom path
- **Tool**: PostCSS (used by Tailwind)
- **Can Move?**: ‚úÖ YES - Can move to `config/postcss.config.js` and reference in `next.config.js`
- **Recommendation**: Keep in root (standard convention, small file)

### 4. **tailwind.config.js**
- **Why**: Tailwind can be configured via `package.json` or custom path
- **Tool**: Tailwind CSS
- **Can Move?**: ‚úÖ YES - Can move to `config/tailwind.config.js` and update `postcss.config.js`
- **Recommendation**: Keep in root (standard convention, small file)

## üîß If You Want to Move Playwright Configs

If you decide to move Playwright configs to `e2e/`:

1. Move files:
   ```bash
   mv playwright.config.ts e2e/
   mv playwright.config.setup.ts e2e/
   ```

2. Update `package.json` scripts (if you have test scripts):
   ```json
   "test:e2e": "playwright test --config e2e/playwright.config.ts"
   ```

3. Update any imports in the config files that reference root paths.
